# Known Issues

Current known limitations and rough edges in the baseline. Tracked engineering **risks** (with
severity, mitigation, owner, status) live in [`docs/RISK_LOG.md`](docs/RISK_LOG.md); this file is
the lighter-weight list of "things that work but you should know about."

Last reviewed: **2026-06-16** (`v0.2.0-sprint1-baseline`).

| ID | Area | Issue | Workaround / notes |
|---|---|---|---|
| KI-01 | Tests | `tests/test_schema.py` are live-DB integration tests that default to `DATABASE_URL` on **port 5432**. With the Docker Compose Postgres (port **5433**), they fail to connect unless `DATABASE_URL` is exported. | Export `DATABASE_URL` before running (the smoke test and CI do this). Unit tests are unaffected. |
| KI-02 | Frontend (dev) | The Turbopack dev server (`next dev`) occasionally emits a fatal HMR/compile panic on first compile of a route, leaving the page blank until restart. | The production build (`next build` / `next start`) is unaffected. Restart `npm run dev`, or use `next start` for stable demos/screenshots. |
| KI-03 | Secrets | `backend/.env` is git-ignored and not in version control, but live API keys may exist in a developer's local copy. | Rotate any keys that were ever shared; keep only the `.env.example` templates in git. CI/deploy should inject secrets, not commit them. |
| KI-04 | Phase 2 | The `StockMovement` ledger module is a **skeleton stub** — its model/service/router raise `NotImplementedError`. Inventory still runs on the Phase 1 lot model. | Intentional for the baseline; implemented in Sprint II per [`docs/architecture.md`](docs/architecture.md). |
| KI-05 | API surface | OpenAPI docs are auto-generated by FastAPI at `/docs` and `/openapi.json`, but there is no exported/published API client or Postman collection yet. | Use the live `/docs` UI. A published contract can follow in a later sprint. |
| KI-06 | Deprecations | `python-jose` triggers a `datetime.utcnow()` deprecation warning under Python 3.13. | Cosmetic; no functional impact. Track for a future jose/datetime update. |

To report a new issue, add a row here (and a risk entry in `docs/RISK_LOG.md` if it needs active
management), or open a ticket and reference it on a `ticket-NN/*` branch.
